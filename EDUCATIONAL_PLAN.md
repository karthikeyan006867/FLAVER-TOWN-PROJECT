## üíæ PROGRESS SAVING & TRACKING

### Saving Mechanism
‚úÖ **Dual-layer Storage:**
1. **Local Storage (localStorage):** Instant, client-side persistence
2. **Clerk API (Server-side):** Secure, cross-device sync

### Tracked Metrics
- ‚úÖ Completed Lessons (per course)
- ‚úÖ Completed Challenges
- ‚úÖ Points Earned
- ‚úÖ Daily Streak
- ‚úÖ Longest Streak
- ‚úÖ Time Spent (total & weekly)
- ‚úÖ Achievements Unlocked
- ‚úÖ Last Study Date

### Sync Flow
```
User Action (Complete Lesson)
    ‚Üì
Update Local State (Zustand)
    ‚Üì
Update localStorage
    ‚Üì
POST to /api/sync-progress
    ‚Üì
Update Clerk Public Metadata
    ‚Üì
GET /api/sync-progress/get (on page reload)
    ‚Üì
Restore user progress
```

### Verification
‚úÖ Endpoint: `/api/sync-progress` - Validates and saves progress
‚úÖ Endpoint: `/api/sync-progress/get` - Retrieves user progress
‚úÖ Rate Limited: 30 requests/minute per user
‚úÖ Authenticated: Requires valid Clerk session
